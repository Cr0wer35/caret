# v0.1 — Correction (streaming ghost text)

_Last updated: 2026-04-21_

## Goal

Ship a working macOS app that displays streamed grammar/typo corrections as the user types, in any app, accept-on-`Tab`, via BYOK Anthropic API.

Dogfoodable by the author daily for 2+ weeks without reverting to ChatGPT/Claude for correction.

## M0 — Project setup

**Goal**: an Xcode project that builds, lints, and launches a hidden menu-bar app.

**Tasks**:

- Create Xcode project `Caret.xcodeproj`, macOS app target.
- `Info.plist`: `LSUIElement = YES`, bundle ID `com.caret.app` (or final chosen ID).
- Enable strict Swift 6 concurrency (`SWIFT_STRICT_CONCURRENCY = complete`).
- Add `NSStatusItem` with a placeholder template icon.
- SwiftLint integrated with a strict config, runs as a build phase.
- `swift-format` configured, pre-commit hook added.
- GitHub Actions: `build` workflow (macOS latest, `xcodebuild build`).
- `CHANGELOG.md` initialized with `## [Unreleased]` section.

**Acceptance**:

- `xcodebuild build` succeeds.
- App launches, no Dock icon, one icon appears in the menu bar.
- SwiftLint produces zero warnings.
- CI is green on `main`.

**Branch**: `m0/setup`.

## M1 — Permissions & onboarding

**Goal**: the user grants Accessibility permission via a one-screen onboarding on first launch.

**Tasks**:

- `Permissions.swift`: wraps `AXIsProcessTrustedWithOptions` and polling.
- `OnboardingView.swift` (SwiftUI): a window with title, one paragraph of explanation, one button that opens `x-apple.systempreferences:com.apple.preference.security?Privacy_Accessibility`.
- Poller: every 1s, checks if permission is now granted; when true, closes the window and transitions to "active" state.
- Menu bar icon: shows a warning state (`exclamationmark.triangle`) until permission is granted.
- Second-launch path: if permission is already granted, skip onboarding.

**Acceptance**:

- Fresh install shows the onboarding window.
- Granting permission in System Settings is detected within 2s.
- Menu bar icon transitions to active state automatically.
- Revoking permission later shows the warning icon again.

**Branch**: `m1/permissions`.

## M2 — Text capture

**Goal**: given a focused text field in any app, Caret can extract (text, cursor range, bounds for range).

**Tasks**:

- `AXHelpers.swift`: thin Swift wrappers around `AXUIElement`, `AXUIElementCreateSystemWide`, `AXUIElementCopyAttributeValue`, `AXUIElementCopyParameterizedAttributeValue`.
- `TextCaptureActor.swift`: exposes `focusedContext(maxChars: Int) async -> FocusedContext?`.
  - Returns `nil` if no focus or secure field.
  - `FocusedContext = (text: String, cursorRange: NSRange, caretScreenRect: CGRect?)`.
- `CGEventTap` listener for `keyDown`, calls into `TextCaptureActor` after a 50ms delay (to let the app process the key).
- Simple debug overlay (togglable via env var `CARET_DEBUG_OVERLAY=1`): a small window showing in real time what Caret is reading.

**App matrix** tested manually during this milestone:

- TextEdit (baseline, AX fully supported)
- Mail, Messages, Notes (Apple apps, AX excellent)
- Safari text field
- Chrome text field
- Slack desktop (Electron — expect degraded)
- Discord (Electron — expect degraded)
- VS Code editor (Electron — expect degraded)
- Notion web (expect mixed)
- Figma web (expect bad — canvas)

**Acceptance**:

- AX read succeeds (text + range + bounds) in at least 7 of 10 test apps.
- Fall-through case identified and logged for each failing app.
- No crash, no hang, no main-thread block.
- `TextCaptureActor` has unit tests for its Swift logic (AX calls mocked).

**Branch**: `m2/text-capture`.

## M3 — Denylist

**Goal**: Caret never reads text from password managers, terminals, or secure fields.

**Tasks**:

- `Denylist.swift`: hardcoded default list of bundle IDs + role-based check.
- Defaults: `com.1password.1password8`, `com.1password.1password`, `com.bitwarden.desktop`, `com.apple.Passwords`, `com.apple.Terminal`, `com.googlecode.iterm2`, `dev.warp.Warp-Stable`, etc.
- Role check: if focused element has `AXSecureTextField` subrole, skip (already enforced in M2a via `AXHelpers.isSecure`).
- Wired into `TextCapture.capture`: denylist check before any text read.
- Unit tests covering: known-bad bundle, known-good bundle, secure role, custom user entry.

**Acceptance**:

- Typing a password in 1Password → no AX read, no LLM call (verified via logs).
- Typing in Terminal → same.
- Macro test: run the app with a test script that simulates keystrokes in a password field — assert zero outgoing requests.

**Branch**: `m3/blacklist` (denylist rename kept to avoid inclusive-language lint failures).

## M4 — Trigger engine

**Goal**: given a stream of `KeyEvent`s, decide when to fire a correction request.

**Tasks**:

- `TriggerEngine.swift`: an actor with:
  - `recordKeystroke(_ event: KeyEvent)` — called from `InputCoordinator`.
  - Internal state: word count since last fire, last keystroke timestamp, pending debounce token.
  - Rules: fire when (≥ 3 new words **or** sentence boundary) AND (≥ 400ms since last keystroke).
- `CancellationToken.swift`: simple `Sendable` struct passed to in-flight LLM calls.
- Decision log (behind a debug flag): prints `[trigger] fire: reason=sentenceBoundary, words=5, idle=412ms`.
- Unit tests: 15+ scenarios including rapid typing, pauses, sentence ends, modifier keys, Tab keys.

**Acceptance**:

- In synthetic tests, the engine fires only when rules match.
- No spurious fires during: mid-word pauses, Cmd+key shortcuts, Tab presses.
- Cancellation token cancels an in-flight task within 50ms.

**Branch**: `m4/trigger-engine`.

## M5 — LLM layer

**Goal**: send a context to Claude Haiku 4.5, stream a JSON structured response, decode it into a `CorrectionResponse`.

**Tasks**:

- `KeychainStore.swift`: store/read/delete API key (Security framework directly, no wrapper).
- `LLMProvider.swift`: the protocol.
- `AnthropicProvider.swift`: implements streaming SSE via `URLSession.bytes(for:)`.
- `SSEParser.swift`: line-by-line parser, yields structured events.
- `CorrectionResponse.swift`: the JSON model.
- `Resources/Prompts/CorrectionPrompt.v1.md`: the system prompt.
- `CorrectionCache.swift`: LRU, 100 entries, 10-min TTL, key = SHA-256(context + model).
- Rate limiter: token bucket, 30 req/min default.
- Error taxonomy: `invalidKey`, `rateLimited`, `offline`, `timeout`, `malformed`.
- Integration test with a mock HTTP server returning canned SSE streams.

**Acceptance**:

- Real request to Anthropic with a test key returns a parsed `CorrectionResponse` for a French typo-laden sentence.
- Cancel mid-stream works (no leaked task, no late UI update).
- LRU cache deduplicates within a session.
- No API key ever logged.

**Branch**: `m5/llm-layer`.

## M6 — UI: suggestion panel

**Goal**: the ghost-text panel appears next to the caret as tokens stream in, accepts on `Tab`, dismisses on any other key.

**Tasks**:

- `SuggestionPanel.swift`: `NSPanel` subclass with `.floating` level, no title, rounded corners, opacity.
- SwiftUI content view: shows streamed text with dimmed foreground + `⇥` hint.
- Positioning logic: reads `caretScreenRect`, places panel 8pt above, flips below near top edge.
- Degraded mode: a pill view anchored top-right of active screen.
- `Tab` interception: CGEventTap callback returns `nil` (swallow) when panel is visible and carries a suggestion.
- Accept handler: replaces `(span_start..<span_end)` with corrected text via `AXUIElementSetAttributeValue` on `kAXSelectedTextRangeAttribute` and `kAXSelectedTextAttribute`.
- Fallback accept: pasteboard + menu bar tooltip "Copied — paste with ⌘V".
- Dismiss handlers: any non-Tab key, focus change, click outside.

**Acceptance**:

- In TextEdit: typing a sentence with a typo shows ghost text; `Tab` replaces the typo in place.
- In Slack (Electron): pill appears at top-right; `Tab` copies to pasteboard + tooltip shown.
- No panel flicker during rapid typing.
- `Tab` without a visible suggestion behaves normally (inserts tab or moves focus).

**Branch**: `m6/ui-suggestion`.

**Acceptance**:
