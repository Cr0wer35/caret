# v0.2 — Autocomplete (directional)

_Last updated: 2026-04-21_

**Status**: Backlog. Not started. Details will be refined after v0.1 ships and receives feedback.

## Goal

Add Copilot-style next-phrase prediction. As the user types, Caret offers the likely continuation of the sentence as ghost text. `Tab` accepts.

This shifts Caret from "fix what I typed" (v0.1) to "predict what I'm about to type" (v0.2). The UX surface is the same panel; the trigger and prompt are different.

## Why v0.2, not v0.1

- v0.1 establishes the entire streaming ghost-text mechanism and proves the UX.
- v0.2 adds a second trigger mode without changing the infrastructure.
- Autocomplete without a proven correction path would be harder to evaluate (noise from typos masks prediction quality).

## Open questions to resolve before starting

1. **Unified suggestion or mode switch?** When the text has a typo, do we correct first and then predict the continuation, or are they two independent modes toggled by context?
2. **Minimum trigger**: predict after 1 word typed? 2? Only mid-sentence?
3. **Prediction length**: next word, next phrase, next sentence? How do we stop the model from over-predicting?
4. **Prompt separation**: does autocomplete reuse `CorrectionPrompt.v1.md` with a different schema, or a new file `AutocompletePrompt.v1.md`?
5. **Apple Intelligence overlap**: on macOS 15.1+, the OS already does some inline prediction in Apple apps. Do we disable Caret autocomplete when Apple's is active in the same field?

## Likely milestones (to be refined)

- **A1** — Second prompt variant + structured output schema for `{predict: string, confidence: float}`.
- **A2** — New trigger mode in `TriggerEngine`: mid-sentence typing, short idle (~200ms), no boundary required.
- **A3** — Mode switcher: decide per-trigger whether it's a correction or a prediction.
- **A4** — UX tuning: prediction text visually differentiated from correction (e.g. different dim level or icon).
- **A5** — Dogfood 2 weeks.
- **A6** — Release v0.2.

## Non-goals

- No per-user fine-tuning or style learning in v0.2.
- No offline cache of prediction patterns.
- No multi-line predictions.

## Risks

- **Flicker**: with two triggers running, suggestions may compete or replace each other. The mode switcher must be decisive.
- **Cost**: more frequent triggers means more tokens. Rate limit must be revisited.
- **Cotypist parity**: Cotypist already does autocomplete. v0.2 will be compared to it directly. Quality bar is "at least as good as Cotypist, but integrated with Caret's correction".
